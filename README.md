# NITC Query Bot 
**Agentic AI for Administrative Support**

An autonomous, locally-hosted Retrieval-Augmented Generation (RAG) backend engineered to answer student queries strictly from the official National Institute of Technology Calicut (NITC) academic regulations.

This project bypasses standard character chunking in favor of a custom regex-based parser (R.x) with metadata injection to prevent semantic drift. It utilizes a hybrid retrieval engine (ChromaDB + SQLite/BM25) and runs 100% locally using Llama 3.2 (3B).

---

##  Prerequisites

Before running this project, ensure you have the following installed:
1. **Python 3.8+**
2. **Ollama:** Download and install from [ollama.com](https://ollama.com/)

---

##  Installation & Setup

**1. Clone the repository**
\`\`\`bash
git clone https://github.com/DIYA-K-GIT/nitc-ai-assistant.git
cd nitc-ai-assistant
\`\`\`

**2. Install Python Dependencies**
Install the required RAG and server libraries:
\`\`\`bash
pip install fastapi uvicorn langchain-huggingface langchain-community chromadb sentence-transformers pymupdf pydantic
\`\`\`

**3. Pull the Local LLM**
Ensure Ollama is running on your machine, then pull the Llama 3.2 model:
\`\`\`bash
ollama pull llama3.2:3b
\`\`\`

---

##  Running the System

To start the bot, you need to run the data ingestion pipeline once, and then start the FastAPI server.

**Step 1: Build the Knowledge Base**
If you have added new PDF regulations, run the pipeline to chunk the PDFs, generate embeddings, and populate ChromaDB and SQLite.
\`\`\`bash
python pipeline.py
\`\`\`

**Step 2: Start the Backend Server**
Start the FastAPI server. This will listen for incoming chat queries and route them through the RAG engine.
\`\`\`bash
python server.py
\`\`\`
*The server will start at: `http://127.0.0.1:8000`*

##  Testing the Widget (Frontend)

This project includes a vanilla JavaScript widget (`widget.js`) designed to be injected into any website globally. 

**To test it locally:**
1. Ensure your backend (`server.py`) is running.
2. Create a basic `index.html` file in the same folder as `widget.js`:
   \`\`\`html
   <!DOCTYPE html>
   <html>
   <head>
       <title>Widget Test</title>
   </head>
   <body>
       <h1>Testing the NITC Query Bot</h1>
       <!-- Inject the widget here -->
       <script src="widget.js"></script>
   </body>
   </html>
   \`\`\`
3. Open `index.html` in any web browser. The floating chat button will appear in the bottom right corner!

##  Architecture Overview
* **`pipeline.py`:** Custom PyMuPDF extractor and regex rule parser. Handles metadata injection and vector/BM25 database population.
* **`server.py`:** The FastAPI backend. Contains the `SYNONYM_MAP`, LLM query rewriter, hybrid search logic, and defensive response parsers.
* **`widget.js`:** Pure JavaScript/CSS floating chat interface with `sessionStorage` memory to persist chat history across page reloads.
* **`chroma_db/` & `college_rules.db`:** The local vector and relational databases (auto-generated by the pipeline).
